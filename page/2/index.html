iOS传值相关（陆续更新）

在iOS开发中，我们有时需要在两个页面中传递参数，

正向传值

我们有两个ViewController一个是发送者MainController 一个是接受者TestViewController

步骤

1.在接受者TestViewController中.h文件中公开声明一个属性

@property(nonatomic,copy) NSString *testLable;

2.在发送者MainController中导入接受者TestViewController的头文件，并实例化一个TestViewController

简单的正向传值就完成了

segue传值

这是一种一种在ViewController类中实现preparedForSegue:sender：的方法，这个方法是在画面切换前就要调用的，因此可以把数据传送到下一个ViewController类中。

预备：两个ViewController组件，
ViewController1中有textfield，button1
ViewController2中有一个button2和label

步骤

1.打开ViewController1.h，将其中的textfield组件用IBOutlet连线，变量名为textbox.button1连线到ViewController2.

2.打开ViewController2.h,将其中的label组件用IBOutlet连线，变量名为label。声明一个变量use_input_data，负责接受ViewController1传来的数据。声明一个方法passdata,用来设置use_input_data.

3.打开ViewController2.m,设置button2的IBAction的方法，将use_input_data赋值给label的text。制作passdata方法，通过这个方法传值。

4.打开ViewController1.m。导入ViewController2头文件，制作preparedForSegue：sender这个方法，将textfeild里面的参数传给ViewController2.

Learn Objective-C first step(nunu原创翻译)

Learn Objective-C first step(nunu原创翻译)
总结
这两天心情有点阴郁，学习效率不是很高。前段时间的熬测被刷，让我感觉到自己对Objective-C的语法的使用还是很生疏的，基础没抓好，以后肯定会很吃亏。现在的我不应该急着去做一份成品app出来，而是要好好反思一下。
Learn Objective-C first step(nunu原创)
1.在对象上调用的方法的基本语法
[object method]
[object methodWithInput:input]
2.方法返回值
output = [object methodWithOutput]
output = [object methodWithInputAndOutput:input]
3.在类中调用方法，创建对象。下例在NSString类中调用了string方法，返回了一个新的NSString对象
id myObject = [NSString string]
Id 类型意味着 myObject 变量可以引用任何类型的对象，动态类型
所有OC中的对象变量都是指针类型
NSString myString = [NSString string]
4.方法的嵌套
在很多语言中方法是这样嵌套的
function1 ( function2())
但是在OC中，方法是这样嵌套的
[NSString stringWithFormat:[prefs format]]
warn: OC中避免一行有两层嵌套
5.多参数的方法
-(BOOL)writeToFile:(NSString)path
atomically:(BOOL)useAuxiliaryFile

一个方法的名称可以分成几段，这里的参数是NSString类型的path和BOOL类型的useAuxiliaryFile
这样调用多参数方法
BOOL result = [myData writeToFile]:@”/tmp/log.txt” atomically: NO]
writeToFile:atomically: 是实例的方法
6.存取器
在OC中所有实例变量默认都为私有的，所以一般你应该使用访问器来获取和设置值。有两种语法。这是传统 1.x 语法
[photo setCaption : @”Day at the Beach”]
output = [photo caption]
第二行代码不直接阅读实例变量，它实际上调用一个方法caption。在大多数情况下，OC中的getter方法不需要加get前缀
每当你看到在方括号内的代码，表示正在向一个对象或一类发送一条消息
7.点语法
(这是在Objective-C 2.0和Mac OS X 10.5的老版本)
photo.caption = @”Day at the beach”
output = photo.caption
点语法只用于set和get方法
8.创造对象
第一种方法
NSString myString = [NSString string]
这种是自动创建类型对象
第二种方法
NSString myString[[NSString alloc] init]
这是一种多重嵌套的方法。第一层的alloc调用的事NSString自己，这是一种相对较低级的调用，能够保存内存并且实例化一个对象
第二层是对初始化的新对象的调用，Init执行通常没有基本的安装步骤，例如创造实例化变量。
有的时候你需要不同版本的init
NSNumber value = [[NSNumber alloc] initWithFloat:1.0]
创建一个NSNumber类的float型数
9.基本内存管理
如果你写给 Mac OS X 的应用程序，您可以选择启用垃圾收集装置。一般情况下，这意味着你不需要考虑内存管理，直到你得到到更复杂的情况。
然而，你不可能总是使用支持垃圾回收的环境。在这种情况下，你需要知道的几个基本概念。
如果你创建使用手动分配样式的对象，之后需要释放的对象。但是，你不应该手动释放 autoreleased 对象，如果你释放了，程序会崩溃
NSString string1 =[NSString string]//自动释放
NSString* string2 =[[NSString alloc] init]
[string2 release]//完成后释放
还有更多了解内存管理，但它将更有意义以后我们看看更多的几个概念
10.创建类
.h文件

#import
@interface Photo : NSObject {
NSString caption;
NSString photographer;
}
-(NSString )caption;
-(NSString )photographer;
-(void) setCaption: (NSString)input;
-(void) setPhotographer: (NSString)input;
}
@end
//@interface是类的声明,类的默认类型是id类型
.m文件

#import “Photo.h”
@implementation Photo
-(NSString) caption {
return caption;
}
-(NSString) photographer {
return photographer;
}

-(void) setCaption: (NSString)input
{
[caption autorelease];
caption = [input retain];
}
-(void) setPhotographer: (NSString)input
{
[photographer autorelease];
photographer = [input retain];
}
@end
//每个set处理两个值，第一是对现有对象的引用，第二个是新的输入的对象
//在垃圾收集环境中，我们只是可以直接设置的新值
我们比对一下两个方法，后面一个set方法可以改成
-(void) setCaption: (NSString)input {
caption = input;
}
但是如果你不能使用垃圾回收，你需要释放旧对象，并保留新的对象。
实际上有两种方式释放引用对象，release 和 autorelease,release直接释放，autorelease过段时间释放。
Autorelease方法是更安全的，因为新的和旧的变量值可以指向相同的对象
11.我们可以创建 init 方法设置实例变量的初始值
-(id) init
{
if ( self = [super init] )
{
[self setCaption:@”Default Caption”];
[self setPhotographer:@”Default Photographer”];
}
return self;
}
//单等号，将超类的self赋给self，让超类做自己的初始化
//if语句验证初始化成功，然后设置默认值
12.Dealloc
Dealloc方法是把对象从内存中删除，是删除子类实例化变量的方法
-(void) dealloc
{
[caption release];
[photographer release];
[super dealloc];
}
最后一行非常重要。必须发送消息 [super dealloc]让超类做它的清理，如果我们不这样做，该对象不被删除，这就是内存泄漏
13.内存管理后续（这部分我不是很能理解，后续在多加练习好了）
OC的内存管理系统称为引用计数，你要做的就是保持跟踪的引用，并运行时实际释放内存
//在简单的术语，你alloc的对象，可能有些地方remain，然后发送一个release每个alloc/remain发送。所以如果你使用alloc一次，然后remain一次，你需要释放两次内存。这是理论的引用计数
在实践中，通常是创建对象只有两个原因
1.为保持它作为实例变量
2.一次性使用函数内部的临时使用
还有一个内存释放的例子
NSNumber value1 = [[NSNumber alloc] initWithFloat:8.75];
[self setTotal:value1];

NSNumber* value2 = [NSNumber numberWithFloat:14.78];
[self setTotal:value2];

[value1 release];

15.property
当我们提前写了caption和author的访问器方法,你可能已经注意到的代码非常简单.而前有可能被推广。
property时Objective-C的特性，使我们能够自动生成的访问器中的新功能。而也有一些其他的好处。让我们将之前但photo类使用属性转换
@property 是声明属性的指示，括号中的@retain指定setter方法保留输入值

@synthesize 指令自动生成的 setter 和 getter 方法。对我们来说，所以我们必须为此类执行dealloc 方法
16.调用nil方法
Objective-C中的nil和其他语言中的null相同，不同的是，你可以调用方法零没有崩溃或引发异常.
如果你在nil返回的对象上调用方法，你会得到nil作为返回值

  • (void) dealloc
    {
    self.caption = nil;
    self.photographer = nil;
    [super dealloc];
    }
    17.Categories
    Categories是Objective-C中最有用的特征之一，从根本上来说，一个category允许你在一个已经存在的类中添加方法而不需要附上子类，或者知道他执行的所有细节。
    这特别有用，因为您可以将方法添加到内置对象中。如果你想要将方法添加到你程序中的实例化的NSString中，你只需要添加一个category。没有必要获取任何对象的时候用一个自定义的子类
    for example
    #import

    @interface NSString (Utilities)

    • (BOOL) isURL;
      @end
      这特别像一个声明对象，不同的是这里没有列出超类，并且在括号中有类别的名称。名称可以是任何你想要的，尽管它需要和里面的方法沟通。
      下面是一个执行文件，记住这不是很好地落实的 URL 检测，只是介绍类别的概念。
      #import “NSString-Utilities.h”
      @implementation NSString (Utilities)

      (BOOL) isURL

      {
      if ( [self hasPrefix:@”http://“] )
      return YES;

        else
  return NO;
}

@end

现在您可以使用此方法在任何 NSString中，
NSString string1 = @”http://pixar.com/“;
NSString string2 = @”Pixar”;

if ( [string1 isURL] )
NSLog (@”string1 is a URL”);

if ( [string2 isURL] )
NSLog (@”string2 is a URL”);

与子类不同，类不能添加实例变量。但是，你可以用category去重写类中已经存在的方法。

Error- Permission denied (publickey).错误详解

欢迎大家入坑 mac上对hexo的坑

如果你在 mac 中配置好了hexo，deploy的时候却一直显示：

Error: Permission denied (publickey). fatal: Could not read from remote repository.

哈哈 ，经过本nunu的大量排除错误，找到了是公钥配置的问题引起的。

这个问题的特征是
. ssh -T idealife@github.com可以成功运行，并显示
Hi hehedalulu， You’ve successfully authenticated, but GitHub does not provide shell access.
. 执行sudo hexo deploy的时候报错。（想死的心都有了吧）

我们细细的看一下终端里面的报错内容

Unhandled rejection Error: EACCES: permission denied, open '/Users/idealife/Developer/blog/db.json' at Error (native)
上面提示db.json文件的写权限的错误…
我们要提高权限 这个涉及到sudo的问题
如果你是在window环境中配置，就会对于权限的认识比较浅。没办法谁让我们是高贵的mac

github 官方文档对Permission denied (publickey)这个错误的解释是： “你个傻叉没事不要用sudo”
而实际上，不用sudo是错误的（你妹的官方文档）

解决方案

原因：生成ssh公钥时，用的命令是 ssh-keygen -t rsa -C “982472770@qq.com”
没有加sudo，生成的公钥是当前用户的，路径是 /Users/mac/.ssh。而sudo hexo deploy命令执行的时候应该会去读取的root用户的公钥，很显然root下还没有对应的公钥信息生成。
解决方法
. 生成ssh公钥时，用命令 sudo ssh-keygen -t rsa -C “982472770@qq.com”
. 提取公钥信息并配置到github中通过sudo cat /var/root/.ssh/id_rsa.pub先显示公钥，再直接复制。
然后就成功啦

mac上搭建hexo2016最新攻略

mac上搭建hexo2016最新攻略

（前言）用github＋hexo搭建博客是一件很炫酷的事情，hexo hexo 每次读起来都有一种黑客的感觉，个人认为每一个程序员都应该拥有一个自己域名的博客，因为这样很酷（zhuang）炫（bi）。我之前的mac上搭hexo特别难受，不知道为什么每一次用git push的时候都是有问题的，现在的博主都太不负责任了，写的都不是完整版，于是我决定造福广大的mac用户，用最新的攻略在mac上搭建hexo。
再强调一遍现在的博主们都太不负责任了，挖坑让新手跳。
某只球真好看。。
预备条件:

  1. 一台高贵的mac
  2. github账户仓库名改成这样格式“yourname.github.io”
  3. TextMate编辑器，因为mac自带的编辑器不好用，文件经常打不开，或者没有权限修改
    现在开始

    1. 环境配置

    1.1 配置ssh密钥
    （ssh密钥是连接github仓库用的配置之前最好先在系统偏好设置–共享（蓝牙旁边）–把远程登录和远程管理勾选上，不然ssh公钥虽然在终端的hexo bash里面能够 能够连接github 但是访问的时候就会出现public key denied这种报错）

打开终端，输入

cd ~/.ssh

如果提示：No such file or directory 说明未配置SSH。
本地生成密钥,输入以下命令

sudo ssh-keygen -t rsa -C "你的邮件地址"

注意命令中的大小写不要搞混。一路点回车，按提示指定保存文件夹，不设置密码。

添加公钥到github上

sudo cat /var/root/.ssh/id_rsa.pub

这个时候key的内容就出来了（sudo是获取当前目录下权限）直接复制公钥
登录github，右上角头像—settings(第六个)—-SSH keys(左边第六个)—-new ssh key(title随便取一个)—-add ssh key（command＋v）
测试ssh公钥
在刚刚的git bash命令行中输入

ssh -T git@github.com

如果出现welcome。。。说明已经配置成功了

设置ssh公钥个人信息

sudo git config --global user.name "输入你的名字（）英文）”

sudo git config --global user.email “输入你的邮箱”

安装git和node.js

因为hexo是基于nodejs的，所以必须安装node.js

安装git
其实xcode自带了git的功能，这里就先不介绍xcode的git功能了
介绍另一种
要安装git得先安装 Homebrew ，打开终端输入以下指令。
ruby -e "$(curl -fsSLhttps://raw.github.com/Homebrew/homebrew/go/install)"

然后安装git

sudo brew install git

安装node.js
第一种方法是在命令行里面输入,不行就尝试第二种方法
sudo brew install node

第二种方法是去node.js的 官网 上下载
然后一路安装，接着我们就可以使用npm指令啦

安装hexo

用npm安装hexo

先cd进一个你想创建hexo的文件夹 输入
sudo npm install -g hexo

这个要等一会，等那个斜杠结束之后再做输入，如果npm太慢的话建议再来一次（不然这边会有问题）

sudo hexo init
然后你可以去你指定的文件夹下面看看有没有hexo

cd 进你的hexo文件夹（这里来一个nunu的友情提示，再命令行里面输入cd＋空格之后，可以直接打开finder把文件夹拖入终端 方便不出错）

sudo npm install
npm install 表示安装当前目录package.json文件中配置的dependencies模块

安装自动部署到GitHub的插件
nunu这里要说一下，新手看hexo的文档和看天书没什么区别，看文档是建立在你已经打好hexo的博客的基础上的
这个也是一个坑，第一次我安装的时候就很纳闷，为什么我没有hexo-deployer-git这个文件夹。某只球用她的windows系统就有 然后我在这里卡了一会，没办法，谁让我们是高贵的mac党呢。
因为Hexo 3.0版本之后需要单独安装发布器插件，检查博客目录的node_modules中有没有hexo-deployer-git文件夹，若没有，哈哈，高贵的mac党们，来执行以下命令安装：

sudo npm install hexo-deployer-git --save

然后，这个－save参数的模块的名字就会出现在package.json的dependencies选项中。

修改站点配置_config文件

从Hexo 3.0开始站点配置文件_config.yml就和之前不一样了，部署类型记得要填写成git,仓库地址要改成自己的，还要添加分支master 对了,某只球还特地强调了要用UTF8的格式哦。
在finder中打开hexo–_config.yml,翻到最下面把内容修改成这样

deploy:
type: git
repository:git@github.com:yourname/yourname.github.io.git(这个yourname改成你自己的名称)
branch: master

这边有几个要特别注意的事项
. 因为markdown语法的特殊性，每一个冒号后面都必须跟上半角空格，连deploy后面都要跟上哦
. 也许有的博客上面type类型上面写的是github。你要相信我，那个是老版的。
. 有的博客上面repo的格式是http这个样子的，你不要相信，我这个就是最新版的，我已经吃了很多亏了。

hexo命令的使用

生成静态页面
刚刚的步骤已经把配置完成了，现在我们执行hexo的命令，打开终端cd进你的hexo目录,输入
sudo hexo generate
本地实现hexo
其实sudo hexo g也是一样的效果，这个意思是初始化目录，生成index.html文件
然后输入

sudo hexo server
这个是生成本地的静态页面,启动后，在浏览器中输入local:4000就可以看到自己博客出事的样子了。
安Ctrl+C停止
部署hexo
最重要的一步来了！！！！！
激动！！！！！！！
sudo hexo deploy
这一步我当年试了几万次都没有实现（虚指，体现博主内心的绝望）deploy就是把你的配置还有你的文章部署到github上面。
如果deploy有问题，你可以来问我我的qq是982472770(问问题之前先发红包表示一下诚意)
deploy成功后，你就可以在浏览器中访问http://yourname.github.io/看到你刚刚调试好的博客啦，是不是很激动
新建文章
现在你已经可以新建一个页面了
命令行cd 进hexo的文件目录中，输入
sudo hexo new “title”
就生成新文章在\source_posts\title.md中
编辑文章
打开新建的文章\source_posts\title.md：在—下面写文章
写好之后再sudo hexo g sudo hexo d 文章就可以push上去了
